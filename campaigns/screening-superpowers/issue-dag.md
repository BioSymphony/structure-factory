## Gate States

No-paid work may run only fixture and validation paths with:

- `max_spend_usd: 0`
- `expected_download_bytes: 0`
- repo fixtures or public-accession metadata only
- no provider mutation
- no restricted runtime install
- no private data
- claim ceiling of `fixture_or_demo` evidence and `candidate` scientific claims

Paid work starts no earlier than W05 and requires an explicit operator gate issue. The gate must name the provider, setup posture, max spend, max runtime, stage contract, progress ledger, immutable code delivery path, expected artifact list, hash/pull checks, cleanup policy, and license/use-context posture.
