# Campaigns

Each campaign folder describes a reusable Structure Factory workflow.

Required sections:

- scientific objective
- scope and non-scope
- issue DAG or wave plan
- inputs
- expected artifacts
- acceptance criteria
- validation commands
- license/capability caveats
- final report shape

Current campaign families:

- `cryoem-raw-to-atomic-report`
- `screening-superpowers`
- `pd-l1-pd1-binder-design`
- `model-comparison`
- `publishable-figure-report`
